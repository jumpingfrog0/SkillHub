## 维护规则

- 本文件只记录入口、状态和 checkpoint 摘要。
- 历史 checkpoint 只追加，不重写。
- `requirement_contract.md` 已创建不代表需求澄清完成；有阻塞项或待确认项时，当前阶段仍为 `需求澄清`。
- 只有收到 `CONFIRM_REQUIREMENT` 且未决事项为空，才允许把当前阶段推进为 `需求契约已确认`。
- checkpoint 必须区分“需求契约草稿已生成”和“需求契约已确认”。
- 技术方案写入 `tech_plan.md`。
- 决策变化写入 `decision_log.md`。
